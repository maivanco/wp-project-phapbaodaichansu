# PhapBaoDaiChanSu Project Context & Guidelines

## Overview
This is a modern, containerized WordPress local development environment. 
- **Application Server**: FrankenPHP embedding Caddy.
- **Database**: MySQL 8.0, accessible via phpMyAdmin.
- **Frontend Build**: Vite + Tailwind CSS v4.

## Project Structure
- `src-configs/themes/default-theme/`: The active custom WordPress theme.
- `src-configs/plugins/`: Custom WordPress plugins.
- `src-configs/configs/`: PHP (`php.ini`) and WordPress (`wp-config.php`) configurations.
- `vite.config.js` / `package.json`: Frontend build configuration located at the project root.
- `.env`: Environment variables (use `env.default` as a template).

## Frontend Guidelines (Vite + Tailwind CSS)
- **Do not** manually edit files inside `src-configs/themes/default-theme/dist/`. These are auto-generated by Vite.
- **Entry point**: `src-configs/themes/default-theme/assets/js/vite.js`. All JS and SCSS/CSS should be imported here or linked properly.
- **Styling**: The project uses Tailwind CSS v4 via the `@tailwindcss/vite` plugin.
- **Development**: The `./start.local` script is used to bring up the Docker containers and automatically start the Vite dev server (`npm run dev`) in the background. The Vite dev server supports Hot Module Replacement (HMR) and is configured to trigger a full page reload when `.php` files change inside the theme.

## WordPress Theme Guidelines
- The active theme is `default-theme`.
- **WooCommerce**: The theme is integrated with WooCommerce, with templates overridden inside the `woocommerce/` folder.
- **ACF**: Advanced Custom Fields is heavily used, as indicated by the `acf-json/` directory syncing field groups.
- **Component separation**: Use the `partials/` and `templates/` folders for modular template parts.

## Common Commands
- `./start.local`: Starts the entire stack (Docker services + Vite dev server).
- `docker compose down`: Stops all services.
- `npm run build`: Generates production frontend assets into `dist/`.
